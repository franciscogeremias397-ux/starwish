# 《银河回信》微信小程序需求开发文档 V1.0

> 文档用途：供 Codex / 开发者直接理解并开发微信小程序 MVP。  
> 项目类型：微信小程序  
> 当前版本：V1.0 正式开发版  
> 更新时间：2026-06-18  
> 核心方向：沉浸式星空心愿社区 / 可探索银河星图 / 公开匿名心愿星

---

## 1. 项目概述

### 1.1 项目名称

**银河回信**

### 1.2 产品一句话描述

用户把心愿、心事、吐槽或想说的话送上星空，一颗流星划过夜空并化作一颗闪耀的心愿星；其他用户可以在可拖动、可缩放的银河星图中探索并点击这些星星，看到来自不同地区的人留下的心愿。

### 1.3 产品核心记忆点

```text
一句话 → 一颗流星 → 一颗星 → 一片可以探索的银河心愿星图
```

### 1.4 产品定位

《银河回信》不是普通留言板，也不是普通许愿工具，而是一个具有沉浸感、仪式感和情绪价值的星空心愿小程序。

用户打开小程序后，会看到：

- 深邃的动态星空
- 银河雾带
- 星座轮廓
- 缓慢移动的星轨
- 安静、神秘、治愈的背景音乐
- 散落在星空中的其他用户心愿星

用户可以：

- 输入一段心愿、心事、吐槽或想说的话
- 点击“送上星空”
- 看到一颗流星划过夜空
- 流星最终停留在某处，化作一颗属于自己的心愿星
- 拖动星空，探索更多区域
- 双指放大缩小星空
- 点击其他用户的心愿星，查看他们留下的话、IP 属地和发送时间

---

## 2. 产品目标

### 2.1 体验目标

用户完成一次心愿投递后，应产生以下感受：

```text
我把一句话交给了星空。
我的心事被宇宙接住了。
我不是一个人在夜里有这些情绪。
原来很多人的愿望，也在同一片银河里闪烁。
```

### 2.2 MVP 目标

V1.0 的目标不是做复杂社交，而是做出一个完整可体验的核心闭环：

```text
动态星空首页
 ↓
输入一句话
 ↓
内容审核
 ↓
流星动画
 ↓
生成公开心愿星
 ↓
进入可探索银河星图
 ↓
其他用户点击查看
```

### 2.3 非目标

V1.0 暂不做以下内容：

- 不做真实天文定位
- 不做真实银河位置计算
- 不做真实星座天象推演
- 不做 AI 回信
- 不做聊天
- 不做好友系统
- 不做点赞评论
- 不做举报功能
- 不做首次发布弹窗
- 不做额外隐私说明补充
- 不做付费功能
- 不做完整后台管理系统
- 不做复杂用户资料系统

## 3. 用户体验主流程

### 3.1 首次进入流程

```text
用户打开小程序
 ↓
进入全屏动态星空首页
 ↓
看到满天繁星、银河、星座轮廓、用户心愿星
 ↓
听到或可手动开启安静神秘的背景音乐
 ↓
可拖动星空探索
 ↓
可双指放大缩小星空
 ↓
可点击其他用户心愿星查看内容
 ↓
可在底部输入自己的心愿
```

### 3.2 发布心愿流程

```text
用户输入一段文字
 ↓
点击“送上星空”
 ↓
前端检查内容为空 / 超长
 ↓
调用云函数 submitWish
 ↓
服务端进行内容安全审核
 ↓
审核通过
 ↓
写入云数据库
 ↓
前端触发流星动画
 ↓
流星划过并停留
 ↓
生成一颗闪耀的用户心愿星
 ↓
提示：你的话，已化作一颗星
```

### 3.3 内容审核失败流程

```text
用户输入内容
 ↓
点击“送上星空”
 ↓
调用云函数 submitWish
 ↓
内容安全审核不通过
 ↓
不写入数据库
 ↓
不触发流星动画
 ↓
提示用户修改表达
```

审核失败提示文案：

```text
这句话暂时不能送上星空，请换一种表达。
```

### 3.4 查看他人心愿流程

```text
用户拖动 / 缩放星空
 ↓
发现一颗更闪耀的心愿星
 ↓
点击心愿星
 ↓
弹出星星详情浮层
 ↓
查看心愿内容、IP 属地、发送时间、星语编号
 ↓
关闭浮层，继续探索星空
```

# 5.2 星空地图交互

## 5.2.1 基本概念

首页星空不是固定背景，而是一张大型可探索星图。

用户可以：

- 单指拖动星空
- 双指放大缩小
- 点击心愿星查看详情

星图中所有元素都属于同一个世界坐标系统：

- 背景星星
- 银河雾带
- 星座线条
- 用户心愿星
- 流星落点

---

## 5.2.2 世界坐标系统

建立一张大型虚拟星图。

建议第一版尺寸：

```js
const STAR_MAP = {
  worldWidth: 5000,
  worldHeight: 8000
}
```

每颗用户心愿星存储世界坐标：

```js
{
  worldX: 1820,
  worldY: 2640
}
```

屏幕坐标由 camera 转换得到：

```js
screenX = (worldX - camera.x) * camera.zoom
screenY = (worldY - camera.y) * camera.zoom
```

---

## 5.2.3 Camera 对象

前端维护 camera 状态：

```js
const camera = {
  x: 0,
  y: 0,
  zoom: 1
}
```

字段说明：

```text
camera.x：当前视口在星图世界坐标中的横向位置
camera.y：当前视口在星图世界坐标中的纵向位置
camera.zoom：当前缩放比例
```

---

## 5.2.4 拖动规则

用户单指拖动时：

```text
手指向左拖动：查看右侧星空
手指向右拖动：查看左侧星空
手指向上拖动：查看下方星空
手指向下拖动：查看上方星空
```

交互要求：

- 拖动要顺滑
- 可加入轻微惯性
- 到达星图边界时有阻尼
- 不得出现纯黑空白边缘
- 星空中的所有元素整体跟随 camera 变化

---

## 5.2.5 缩放规则

用户双指缩放时：

```text
双指放大：靠近星空，心愿星更明显
双指缩小：远看银河，看到更大范围星空
```

建议缩放范围：

```js
const MIN_ZOOM = 0.6
const MAX_ZOOM = 2.5
const DEFAULT_ZOOM = 1
```

缩放要求：

- 以双指中心为缩放中心
- 星星位置保持自然
- 心愿星点击区域随缩放适配
- 详情浮层不随星图缩放，而是固定在页面上层

---

## 5.2.6 边界处理

星图第一版是有限大星图。

建议：

```text
worldWidth: 5000
worldHeight: 8000
```

边界处理：

- 拖到边缘时有轻微阻尼
- 不要突然卡死
- 不要露出空白区域
- 必要时可在边缘继续绘制少量背景星星作为缓冲

# 5.3 星空视觉系统

## 5.3.1 整体风格

关键词：

```text
深色
高级
神秘
安静
治愈
浪漫
宇宙感
空灵
梦幻
```

不要做成：

- 普通留言墙
- 低质动态壁纸
- 花哨霓虹小游戏
- 高饱和赛博风
- 太卡通的儿童风

---

## 5.3.2 背景色

建议使用深色渐变：

```text
顶部：#050711
中部：#0B1024
底部：#101936
```

辅助色：

```text
银河紫：#6C63FF
星光白：#F4F7FF
淡金星光：#FFE8A3
流光蓝：#9DBBFF
星云粉：#FFD6F1
```

---

## 5.3.3 背景星星

背景星星用于营造星空氛围，不承载用户内容。

特点：

```text
不可点击
颜色偏冷
亮度较低
闪烁幅度较小
数量较多
大小随机
用于构成满天繁星
```

视觉建议：

```text
颜色：冷白色 / 淡蓝色
大小：0.5px - 2px
透明度：0.25 - 0.65
光晕：无或极弱
```

背景星星分三层：

### 远景星

```text
数量：根据地图区域生成
大小：0.5px - 1.2px
亮度：低
移动速度：极慢
闪烁幅度：轻微
```

### 中景星

```text
数量：中等
大小：1px - 2px
亮度：中
移动速度：慢
闪烁幅度：中
```

### 近景星

```text
数量：较少
大小：2px - 3px
亮度：较高
移动速度：稍明显但仍然缓慢
闪烁幅度：较明显
```

---

## 5.3.4 用户心愿星

用户心愿星承载用户发布的内容，必须与背景星星明显区分。

特点：

```text
可点击
颜色更明显
亮度更高
有呼吸闪耀动画
有柔和光晕
数量可以不断增加
分布在整片可探索星空中
```

视觉建议：

```text
颜色：淡金色 / 星云紫 / 流光蓝 / 浅粉白
大小：2.5px - 5px
光晕：明显但柔和
闪烁：呼吸式闪耀
点击范围：星点周围扩大 16px - 24px
```

心愿星不能过度刺眼，仍然要保持安静、治愈的氛围。

---

## 5.3.5 银河效果

第一版不依赖真实天文图片，可使用 Canvas 生成艺术化银河。

银河是一条柔和的星云带。

建议方向：

```text
从左上到右下
或从左下到右上
```

效果要求：

```text
颜色：蓝紫、淡白、微粉
透明度：低
形态：柔和、弥散、不规则
运动：非常缓慢漂移
```

实现建议：

- 沿一条弧形路径生成大量半透明粒子
- 粒子大小随机
- 粒子透明度随机
- 每帧轻微偏移
- 不要闪烁过强
- 不要挡住用户阅读文字

---

## 5.3.6 星座轮廓

第一版展示艺术化星座轮廓，不需要真实天文计算。

要求：

```text
线条颜色：rgba(180, 210, 255, 0.18)
星座星点颜色：rgba(230, 240, 255, 0.65)
线条宽度：0.5px - 1px
出现方式：淡淡浮现
亮度不可过高
```

建议第一版放置 6-8 个常见星座轮廓：

```text
白羊座
金牛座
双子座
巨蟹座
狮子座
处女座
天秤座
天蝎座
```

星座数据示例：

```js
{
  name: 'Leo',
  label: '狮子座',
  stars: [
    { x: 0.68, y: 0.22 },
    { x: 0.72, y: 0.25 },
    { x: 0.75, y: 0.31 }
  ],
  lines: [
    [0, 1],
    [1, 2]
  ]
}
```

---

## 5.3.7 星轨运动

星轨运动要非常慢，给用户一种“夜空在缓慢流动”的感觉。

建议：

```text
所有背景星星围绕中心点进行极慢旋转
旋转速度：每帧 0.00001 - 0.00003 弧度
叠加轻微漂移
```

注意：

- 不要太快
- 不要影响用户点击
- 不要影响文字阅读
- 用户心愿星可以有轻微呼吸闪耀，但不要跟随过强旋转导致坐标难以点击

# 5.4 流星动画

## 5.4.1 触发条件

用户输入合法内容并通过内容安全审核后，触发流星动画。

注意：

```text
审核不通过时，不触发流星动画。
```

---

## 5.4.2 动画流程

```text
1. 用户点击“送上星空”
2. 按钮变为“送往银河中……”
3. 内容审核通过
4. 输入区轻微淡出或下沉
5. 星空整体轻微变暗 / 呼吸
6. 流星从当前视口边缘进入
7. 流星拖着尾焰划过
8. 流星逐渐减速
9. 停留在当前可见星空某处
10. 爆出柔和光晕
11. 化作一颗新的用户心愿星
12. 显示提示：你的话，已化作一颗星。
```

---

## 5.4.3 流星轨迹

流星起点从当前屏幕边缘随机出现：

```text
左上
右上
左侧
右侧
```

终点生成在当前视口可见区域内，并避开：

```text
底部输入框区域
顶部按钮区域
屏幕边缘
```

终点范围建议：

```text
x: 当前屏幕宽度 15% - 85%
y: 当前屏幕高度 15% - 65%
```

---

## 5.4.4 流星视觉

```text
头部：明亮光点
尾巴：渐隐长尾
颜色：白色 / 淡蓝 / 淡紫
速度：开始较快，结束变慢
持续时间：1.8 - 2.8 秒
结束光晕：0.8 秒
```

---

## 5.4.5 新星生成

流星结束后，在终点生成新的用户心愿星。

新星特点：

```text
比背景星更亮
有柔和光晕
有呼吸闪烁
可点击
保存为世界坐标
进入公共星空
```

坐标转换示例：

```js
const newStar = {
  worldX: camera.x + meteorEndScreenX / camera.zoom,
  worldY: camera.y + meteorEndScreenY / camera.zoom
}
```

```text
深色半透明背景
轻微模糊

淡蓝紫描边
文字清晰

```

# 5.6 我的星星页面

## 5.6.1 页面路径


## 6. 内容安全需求

### 6.1 审核目标

由于用户发布的心愿会进入公共星空，因此所有用户提交内容必须先经过内容安全审核。

审核不通过时：

```text
不触发流星动画
不写入数据库
不进入公共星空
提示用户换一种表达
```

---

### 6.2 禁止内容范围

以下内容禁止发送：

```text
敏感政治内容
色情低俗内容
赌博相关内容
毒品相关内容
暴恐血腥内容
违法交易内容
诈骗引流
联系方式引流
辱骂攻击
隐私泄露
未成年人不适宜内容
自残自杀诱导内容
其他违反平台规则或法律法规的内容
```

---

### 6.3 审核流程

```text
用户点击“送上星空”
 ↓
前端校验字数和空内容
 ↓
调用云函数 submitWish
 ↓
服务端调用内容安全能力
 ↓
通过：写入数据库并返回新星数据
 ↓
不通过：返回审核失败
```

### 6.4 审核失败提示


不要提示具体违规词，避免用户刻意绕过审核。

## 9. 星图分区加载

### 9.1 分区目的

整片星空可以拖动和缩放，不能把所有心愿星一次性渲染到 Canvas 上。

注意：这里不是限制心愿星总数，而是为了性能按当前视口附近区域加载。

---

### 9.2 Tile 分区

将整张星图按固定大小划分为多个 tile。

示例：

```js
const TILE_SIZE = 500
const tileX = Math.floor(worldX / TILE_SIZE)
const tileY = Math.floor(worldY / TILE_SIZE)
```

每颗心愿星根据 worldX / worldY 归属到对应 tile。

---

### 9.3 加载范围

前端根据当前 camera 计算视口对应的世界坐标范围。

示例：

```text
当前视口区域：
x: 1000 - 2000
y: 1500 - 2500

加载缓冲区域：
x: 800 - 2200
y: 1300 - 2700
```

实际加载：

```text
当前视口 tile + 周边一圈 tile
```

这样用户拖动时，周边星星能提前出现，不会突然空白。

---

### 9.4 渲染规则

```text
当前视口范围内的心愿星显示
视口外的心愿星不渲染
用户拖动到对应区域后再显示
不做固定 80 颗上限
不做强制密度控制
```

如某一区域星星很多，可只做 Canvas 性能层面的批处理，不从产品层面限制展示密度。

### 10.1 submitWish

功能：

```text
提交用户心愿
进行内容安全审核
生成星星数据
写入数据库
返回新星信息
```

入参：

```js
{
  content: "希望今年可以顺利一点",
  worldX: 1820,
  worldY: 2640
}
```

服务端处理：

```text
1. 获取用户 openid
2. 校验 content 长度
3. 调用内容安全审核
4. 审核失败则返回 rejected
5. 解析 IP 属地
6. 生成 contentPreview
7. 计算 tileX / tileY
8. 随机生成 size / brightness / color / constellation
9. 写入 wish_stars
10. 返回新星数据
```

返回示例：

```js
{
  success: true,
  star: {
    id: "star_id",
    content: "希望今年可以顺利一点",
    createdDate: "2026.06.17 23:48",
    worldX: 1820,
    worldY: 2640,
    size: 3.4,
    brightness: 0.95,
    color: "#FFE8A3",
    constellation: "狮子座",
    ipRegion: "湖北"
  }
}
```

审核失败返回：

```js
{
  success: false,
  code: "CONTENT_REJECTED",
  message: "这句话暂时不能送上星空，请换一种表达。"
}
```

## 17. 关键交互细节

### 17.1 进入首页

- 星空立即出现
- 1 秒内主标题和输入区淡入
- 背景星星开始闪烁
- 银河微微流动
- 用户心愿星从云端加载后逐渐浮现

---

### 17.2 拖动星图

- 拖动时星图跟随手指移动
- 拖动结束后可有轻微惯性
- 触达边界时有阻尼感
- 拖动时不误触心愿星
- 短点击才触发星星详情，拖动不触发点击

---

### 17.3 缩放星图

- 双指缩放以手指中心为缩放中心
- 缩放过程中星星大小可适度变化
- 心愿星光晕不宜过度放大
- 缩放后重新计算可见 tile

---

### 17.4 点击心愿星

- 心愿星点击范围要比视觉星点更大
- 建议扩大 16px - 24px
- 点击后显示浮层
- 浮层出现时背景星空仍可轻微动态，但不要干扰阅读

---

### 17.5 发布心愿

- 点击发送后按钮禁用
- 审核期间显示“送往银河中……”
- 审核通过才开始流星动画
- 动画期间避免重复提交
- 动画结束后清空输入框
- 新星立即加入当前星图缓存

---

## 18. 性能要求

### 18.1 Canvas 性能

要求：

```text
普通设备上拖动不卡顿
缩放不卡顿
流星动画顺滑
首页加载时间尽量短
```

建议：

- 背景星星使用固定随机种子生成，避免每帧重建全部数据
- 心愿星按 tile 缓存
- 只渲染当前视口及缓冲区域
- 减少复杂阴影的使用
- 对远景星做批量绘制
- 拖动过程中降低部分高成本光效
- 停止拖动后恢复完整光效

---

### 18.2 网络请求

拖动和缩放过程中，不要每帧请求云函数。

建议：

```text
camera 变化后 debounce 200-400ms
只在进入新 tile 区域时请求
已经加载过的 tile 使用本地缓存
```

```text

不公开手机号、微信号等敏感信息
```

## 20. V1.0 验收标准

完成后必须满足以下标准：

### 20.1 星空视觉

- 打开小程序能看到全屏动态星空
- 星空包含背景星星、银河、星座轮廓
- 星星会缓慢闪烁
- 银河有轻微流动感
- 整体视觉安静、神秘、深邃、治愈
- 不像普通留言墙
- 不像低质动态壁纸

### 20.2 星图交互

- 星空可以单指拖动
- 星空可以双指放大缩小
- 拖动和缩放流畅
- 星图有边界阻尼
- 拖动时不误触星星
- 缩放后心愿星位置正确

### 20.3 心愿星

- 用户心愿星与背景星星明显区分
- 背景星星不可点击
- 用户心愿星可点击
- 用户心愿星有呼吸闪耀动画
- 点击心愿星显示详情浮层
- 详情显示心愿内容、IP 属地、发送时间、星语编号
- 不展示完整 IP、openid、头像、昵称

### 20.4 发布流程

- 用户可以输入一句话
- 空内容不能发送
- 超过 80 字不能发送
- 点击发送后调用内容安全审核
- 审核通过后触发流星动画
- 流星动画结束后生成新心愿星
- 新心愿星保存到数据库
- 新心愿星出现在公共星空
- 审核不通过时不触发流星动画

### 20.5 我的星星

- 我的星星页面能显示当前用户发布记录
- 点击记录可查看详情
- 用户可以删除自己的星星
- 删除后公共星空不再显示该星星

### 20.6 音乐

- 有背景音乐开关
- 可播放 / 暂停
- 可循环播放
- 记住用户上次设置
- 不影响核心功能

### 20.7 明确不做

- 不做举报功能
- 不做首次发布弹窗
- 不做额外隐私说明补充
- 不做 AI 回信
- 不做聊天
- 不做点赞评论
- 不做付费
- 不做真实天文计算

## 22. 给开发者的核心提醒

这个项目的重点不是功能堆叠，而是氛围感。

请优先保证：

```text
星空好看
拖动顺滑
缩放自然
音乐舒服
流星动画有仪式感
心愿星有“被看见”的感觉
点击他人星星时有一点安静的情绪触动
```

不要把它做成普通留言板。

最终体验应该像：

> 用户在一片可以探索的银河里，偶然听见了别人的心事，也把自己的心愿留在了夜空中。
